Leetcode 141 Linked list cycle

题目描述：

Given a linked list, determine if it has a cycle in it.

To represent a cycle in the given linked list, we use an integer pos which represents the position (0-indexed) in the linked list where tail connects to. If pos is -1, then there is no cycle in the linked list.

大致来说就是判断一个单向链表中是否有环的存在，也就是链表中的尾节点，借用leetcode官方解答https://leetcode-cn.com/problems/linked-list-cycle/

中一个例子，举例来说，最后的节点指向了：

通用的解法有两种，一种是快慢双指针法，还有一种是利用哈希表存储链表中每一个node的地址，如果存在cycle的话，通过遍历整个列表就会发现tail ndoe 所指向的node地址已经存在在哈希表中

方法一 哈希表：

时间复杂度:O(n),显然此方法需要遍历整个链表

空间复杂度:O(n)

方法二 快慢双指针法

我们初始化两个指针，分别指向head 与head.next,通过控制两个指针步进的长度，若是存在环，那么最终快指针会与慢指针相遇。

时间复杂度:

链表中不存在环:

递进步长较大的指针会先到达链表尾部，与遍历链表类似，时间复杂度仍然取决于链表长度，为O(n)

链表中存在环：

可以类比小学奥中的火车追赶问题，或者将较慢的指针作为坐标系。无论怎样其实都是速度较快的指针在追赶速度较慢的指针。那么追赶所需的时间就是相差的一圈路程(即链表长度n)与相对速度的比值：
$$
t=(\frac{n}{Vf-Vs})=(\frac{n}{2-1})=n
$$

空间复杂度:

这里只是申请了两个指针的节点空间，空间复杂度为O(1)
