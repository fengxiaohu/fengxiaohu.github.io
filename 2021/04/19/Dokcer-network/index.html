容器之间的通信网络可以分为单个物理机上容器之间的通信与不同物理机之间溶剂的通信。

单个物理机内docker 容器间通信

  • bridge 模式 在计算机网络中，网桥bridge通常是指工作在链路层的设备，在链路层上进行进行frame的转发。网桥可以看作是一个多端口的集线器，只有符合一定的转发规则才会被转发到目的端口。连接在同一个网桥上的容器时可以互相通信的。不同网桥之间通常是不能直接进行通信的。并且网桥也仅适用于在单个物理机内的docker之间进行网络上的通信。

  • 网桥在创建时有两种模式，一种是默认桥接，另外一种是用户定义的桥接。默认桥接方式仅允许dockers之间通过ip地址进行通信。而用户定义的网络模式下可以直接通过容器名称通信。更为详细的不同可以参考官网这里的解释

  • 断开链接

  • Host 模式

    如果使用此模式，那么容器将不会创建自己的网络栈，仅仅会和Host物理机共享一个网络栈。使用时只要指定参数–network host就可以

    Host模式的优势在于可以保障某个docker进程获得网络性能上的优势。因为相比于桥接模式不需要做NAT，不会出现性能损失

    • 这种模式下的容器没有隔离的 network namespace
    • 容器的 IP 地址同 Docker host 的 IP 地址
    • 需要注意容器中服务的端口号不能与 Docker host 上已经使用的端口号相冲突
    • host 模式能够和其它模式共存

  • None 模式

    禁用容器的网络功能 容器只有一个127.0.0.1的loopback地址。

    None模式的优势在于不会对容器的网络配置做事先的约束，换句话说，用户可以在这个干净的容器上做任何关于网络方面的定制化开发

  • Container 模式

    在这个模式下新创建的容器会共享一个网络栈，这样两个容器之间可以通过localhost快速通信
