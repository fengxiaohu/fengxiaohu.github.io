container VS virtual machine

容器与虚拟机一样，终极目的都是为了提高计算资源的使用效率，同时提供应用一个独立的运行单元。

容器与虚拟机的区别，从他们的架构图中我们可以轻而易举地看出不同来

虚拟机实际上拥有自己完整的OS，在设计之初，为了消除对于底层硬件的依赖，我们在 OS 与 底层硬件之间再进行一层抽象，抽象出单独的一层hypervisor。hypervisor使得一台真正的物理机可以在在某种程度上被切分更小的部分，并且切分后不同的虚拟机之间可以运行不同的OS，互相不会被干扰。相比较容器来说，由于每个虚拟机都有自己独立的OS内核，理论上不同虚拟机之间的应用是严格隔离的。但是由于硬件虚拟化的过程中会有一部分性能损失，毕竟你要牺牲一部分的算力来执行硬件虚拟化的过程。

容器相较与虚拟机来说是一种更加轻量化的虚拟化形式。容器通过Cgroup + Namespace实现资源控制与访问隔离。同时同一台物理机上的进程都共享同一个OS，因此隔离程度没有虚拟机那么强。但是好处就是容器的启动与切换速度更快，同时容器基本没有虚拟化过程中的性能损失。
