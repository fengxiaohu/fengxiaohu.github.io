Leetcode 155 Min Stack

题目描述：

Design a stack that supports push, pop, top, and retrieving the minimum element in constant time.

  • push(x) – Push element x onto stack.
  • pop() – Removes the element on top of the stack.
  • top() – Get the top element.
  • getMin() – Retrieve the minimum element in the stack.

根据题目可以看出，要求在常数时间内拿到栈中最小元素的值。那么在栈内排序的思路就行不通了，需要采用额外的空间来降低时间复杂度，可以再申请一个辅助栈，栈顶用来存储最小的元素值。

方法一：辅助栈与数据栈保持同步

时间复杂度：

O(1)，“pop”、“top”、“Getmin”的操作不论数据规模多大，都只是有限个步骤，因此时间复杂度是：O(1)。

空间复杂度：O(N)

这里 N是读出的数据的个数。

方法二：辅助栈与数据栈不保持同步

入栈元素仅比辅助栈顶元素小的时候，压入辅助栈，同样，出栈的时候只有栈顶元素相同时才弹出。

相比较于出入栈都要保持严格同步的方法，方法二会节省一部分辅助栈的空间开销

时间复杂度O(1)

空间复杂度O(n)
