Leetcode 169-Majority-Element

description

Given an array of size n, find the majority element. The majority element is the element that appears more than ⌊ n/2 ⌋ times.

You may assume that the array is non-empty and the majority element always exist in the array.

题目目的就是在数组中找出出现频率最大的元素

Solution

  • 暴力解法

面对此问题时，显而易见的暴力解法就是遍历整个数组并记录每个元素出现的次数，时间复杂度为O(N^2).

  • 哈希表

为了降低时间复杂度，可以利用哈希表来存储数组中元素的哈希值，然后遍历哈希表中键值对，统计出现的频率。对于哈希表具体的原理以及实现，后续我会专门单独写一篇博客来介绍。

由于题目中定义多数元素的数量超过了n/2，因此实际上可以通过迭代的方式来更新出现频率最大的元素

排序法

由于我们所寻找的的众数元素在数组中所占到的数目占到了一半以上，因此我们可以先将数组排序，排序好后数组下标最中间的元素就是我们所寻找的众数元素。Java自带的排序算法是基于快速排序改进的，因此时间复杂度为
$$
O(N\log(N))
$$

随机法

由于我们所寻找的众数在数组元素中占绝大多数，因此从直观上来讲，我们随机挑选一个出来就很有可能是我们所要寻找的目标元素。算法具体实现如下。此算法在最好的情况下，第一次就寻找到众数，时间复杂度为
$$
O(1)
$$
在最差的情况下,会存在一直无法寻找到目标元素的情况，此时时间复杂度为
$$
O(\infin)
$$
面对这种不确定性问题的时候，我们通常都要使用期望来衡量我们所期望时间复杂度。为了方便计算，我们对这个问题做一个优化，假设众数元素恰好占据了数组元素的一半，那么数量超过一半的情况所需要的时间复杂度期望显然要小于恰好为一半的情况
$$
E_{prob}<E_{std}\
E_{std}(1) = 1\times\frac{1}{2}\
E_{std}(2) = 1\times\frac{1}{2}+2\times\frac{1}{2^2}\
…\
E_{std}(n) = \lim {n \rightarrow \infty} \sum{i=1}^{n} i \cdot \frac{1}{2^{i}}=2
$$

分治法

我们将目标数组分成左右两部分，那么其中一个子数组的众数必然是目标数组的众数。通常来说，分治法解决问题可以根据以下三个步骤来解决：

  • 分解：将目标问题划分为子问题
  • 解决：通过递归的方式解决子问题
  • 合并：最后合并子问题的解

在使用分治法来解决此类问题的时候，计算时间复杂度是通过主定理来进行的，有关主定理的介绍可以参考https://brilliant.org/wiki/master-theorem/

最终得到我们的时间复杂度为
$$
\begin{aligned}
T(n) &=\Theta\left(n^{\log _{b} a} \log n\right) \
&=\Theta\left(n^{\log _{2} 2} \log n\right) \
&=\Theta(n \log n)
\end{aligned}
$$

Boyer-Moore 投票算法

这种算法通过维护一种一个候选元素(candidate_number),这里我们选用数组最开始的元素啊array[0]来作为候选元素，然后遍历整个数组，当遇到相同的元素，选票数目count+1，当遇到相反元素count-1,若count为0，那么我们更换这个候选元素。由于只是需要遍历一遍数组且只是维护一个常数变量count，因此时间复杂度是O(n),空间复杂度为O(1)。

Leetcode 155 Min Stack

题目描述：

Design a stack that supports push, pop, top, and retrieving the minimum element in constant time.

  • push(x) – Push element x onto stack.
  • pop() – Removes the element on top of the stack.
  • top() – Get the top element.
  • getMin() – Retrieve the minimum element in the stack.
